# Documentation

This directory contains project documentation, design documents, and user guides.

## Structure

- Architecture documentation
- Design documents and specifications
- User guides and tutorials
- API documentation (supplementary to `/api`)
- Development guides
- Operational runbooks

## Purpose

Centralized location for all project documentation to support:
- Developer onboarding
- System understanding
- Design decisions and rationale
- Operational procedures
- User guidance

## Formats

Documentation can be in various formats:
- Markdown files (`.md`)
- Architecture diagrams
- Sequence diagrams
- API documentation
